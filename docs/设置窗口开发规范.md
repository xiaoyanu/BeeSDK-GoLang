# Bee Go 设置窗口开发规范

## 当前实现

设置窗口位于 Go worker 进程，不进入 Bee C 壳 DLL。当前实现文件：

- `settings.go`：布局常量、DPI 尺寸和设置模型。
- `settings_window_windows.go`：Windows worker 的 Win32/GDI+ 窗口实现。
- `platform_stub.go`：非 Windows 测试桩。

## 行为约定

- 默认客户区：640×420，按系统 DPI 等比缩放。
- 使用标准固定尺寸窗口样式：标题栏、系统菜单、最小化按钮。
- 不使用 `WS_THICKFRAME`、`WS_SIZEBOX` 或最大化按钮。
- 首次打开居中。
- 重复点击设置菜单只恢复并聚焦已有窗口，不创建第二个窗口。
- 禁用和卸载会向窗口线程发送关闭消息，并等待线程真正退出。
- GDI+ 负责背景、卡片、色块和状态圆点绘制。
- GDI `DrawTextW` 在 GDI+ 绘制完成后统一执行，避免文字被后续图形覆盖。
- 中文文本使用 Unicode `WCHAR` 绘制，字体优先 `Microsoft YaHei UI`。

## 生命周期注意事项

不要在 `DllMain` 或 Bee C 壳中创建设置窗口。窗口只能由 Go worker 的 `settings` 事件打开；窗口线程退出后才允许 worker 结束。

禁用/卸载清理顺序：

```text
设置 closing 标志
→ PostMessage(WM_CLOSE)
→ 窗口线程 DestroyWindow/PostQuitMessage
→ 等待消息循环退出
→ worker 继续停止
```

## 真实 Windows 验收清单

WSL 只能验证 Windows/386 编译，以下项目必须在 Bee 中人工确认：

- [ ] 设置菜单能打开窗口
- [ ] 窗口居中
- [ ] 重复点击只存在一个窗口并聚焦
- [ ] 标题、状态、说明文字可见
- [ ] 中文显示正常
- [ ] 标题栏可拖动
- [ ] 四边四角不能缩放
- [ ] 最小化、关闭正常
- [ ] 禁用/卸载后窗口消失
- [ ] 卸载后 worker 进程退出
- [ ] 重新加载插件后仍可再次打开设置窗口
- [ ] 高 DPI 下无裁切或重叠

编译成功不等于 UI 验收成功。
