# Bee 设置窗口设计规范

本文定义 Bee Go 插件开发模板的设置窗口目录、窗口行为、视觉风格、绘制顺序和生命周期约束。新增或修改设置界面时必须遵守本规范。

## 1. 技术与目录

设置窗口采用：

```text
C + Win32 + GDI+ + DWM
```

目录固定为：

```text
settings.go                         # Go 调用入口，不放 UI 实现
settings_window/
├── settings_window.c              # 窗口、消息循环、绘制和交互
└── settings_window.h              # BeeSettingsOpen / BeeSettingsClose
```

规则：

- 窗口相关代码不得放入 `other/`。
- `other/` 只保存 Bee ABI 桥接、导出定义等固定底层文件。
- 不引入 Qt、WebView 或项目私有运行库。
- 最终发布物仍为单个 PE32/i386 DLL。

## 2. 窗口框架规范

窗口保留系统框架样式供 DWM 识别：

```c
WS_OVERLAPPED | WS_CAPTION | WS_SYSMENU | WS_MINIMIZEBOX
```

但可见非客户区必须移除：

```c
case WM_NCCALCSIZE:
    return 0;
```

这样客户区覆盖整个窗口，不显示系统标题栏及其顶部、左侧边线。

系统阴影使用：

```c
MARGINS margins = {1, 1, 1, 1};
DwmExtendFrameIntoClientArea(hwnd, &margins);
```

禁止以下做法：

- 不要仅通过 `DWMWA_BORDER_COLOR` 伪装边框颜色；它不能可靠消除标题栏下沿和固定非客户区边线。
- 不要使用 `CS_DROPSHADOW` 代替 DWM 阴影。
- 不要在客户区自绘外层窗口阴影。
- 不要添加 `WS_THICKFRAME` 或 `WS_MAXIMIZEBOX`。

## 3. 固定尺寸与拖动

设置窗口是固定尺寸窗口：

- 不允许拖动四边或四角缩放。
- 不显示最大化能力。
- 初次打开时按 Windows 工作区水平、垂直居中。
- 重复点击设置时聚焦已有窗口，不创建第二个实例。

顶部自绘标题区域通过命中测试支持拖动：

```c
case WM_NCHITTEST:
    // 标题区空白返回 HTCAPTION
    // 关闭按钮等交互区域返回 HTCLIENT
```

要求：

- 标题区空白可拖动窗口。
- 关闭按钮不能被拖动逻辑覆盖。
- 客户区其他内容不应误触发拖动。
- 保留系统窗口样式，以维持 DWM 阴影及系统窗口管理兼容性。

## 4. 视觉规范

整体风格：简约、现代、低噪声、浅色。

当前基础设计令牌：

| 用途 | 值 |
|---|---|
| 客户区背景 | `#F6F7FB` |
| 卡片背景 | `#FFFFFF` |
| 主文字 | `#182230` |
| 次文字 | `#667085` |
| 分隔线 | `#E4E7EC` |
| 主色 | `#6366F1` |
| 主色悬停 | `#5558E8` |
| 主色按下 | `#484BD6` |
| 成功色 | `#12B76A` |

布局规则：

- 使用统一 DPI 缩放，不写死仅适用于 96 DPI 的尺寸。
- 外边距、卡片内边距、控件间距保持一致。
- 卡片和按钮使用适度圆角。
- 不给白色卡片绘制模拟窗口阴影。
- 卡片可使用轻量分隔线，但不使用明显深色描边。
- 自绘关闭按钮置于右上角，具备悬停反馈。
- 主按钮具备默认、悬停、按下三种状态。

## 5. 绘制顺序

窗口必须使用双缓冲，避免闪烁：

```text
WM_PAINT
→ CreateCompatibleDC / CreateCompatibleBitmap
→ GDI+ 绘制背景、卡片、圆角、按钮和图标
→ 释放 GDI+ Graphics
→ Win32 DrawTextW 绘制全部文字
→ BitBlt 到窗口
```

关键约束：

- `WM_ERASEBKGND` 返回 `1`。
- 文字必须在全部 GDI+ 图形完成后统一绘制。
- 不要把 `DrawTextW` 穿插在仍活动的 GDI+ 绘制流程中，否则后续 GDI+ 刷新可能覆盖文字。
- 中文文字使用 Unicode `WCHAR`/`DrawTextW`。
- 字体优先使用 `Microsoft YaHei UI`，创建失败时回退到 `DEFAULT_GUI_FONT`。
- 按钮、状态标签和标题文字均必须在真实窗口截图中可见，不能只以“编译成功”作为 UI 验证。

## 6. 控件策略

适合 GDI+ 自绘：

- 卡片
- 按钮
- 状态标签
- 图标背景
- 分隔线
- 简单开关和进度状态

复杂输入控件优先使用原生 Win32 子控件：

- 文本输入：`EDIT`
- 多行文本：`EDIT | ES_MULTILINE`
- 下拉选择：`COMBOBOX`
- 列表：`LISTBOX` 或 `LISTVIEW`

不要自行重写输入法、光标、选择、复制粘贴和无障碍行为。

## 7. 线程与生命周期

- 设置窗口运行在独立 UI 线程，不阻塞 Bee 设置入口。
- GDI+ 在 UI 线程启动一次、退出时关闭一次。
- 禁止在 `DllMain` 中创建窗口、线程或初始化 GDI+。
- 插件禁用和卸载时调用 `BeeSettingsClose`。
- 关闭窗口后 UI 线程必须退出。
- DLL 卸载前必须等待窗口线程结束，避免窗口过程指向已卸载代码。

## 8. 验收清单

每次修改窗口后至少检查：

- [ ] 窗口初次打开居中
- [ ] 重复打开只聚焦已有窗口
- [ ] 顶部空白区域可以拖动
- [ ] 关闭按钮可点击且不触发拖动
- [ ] 四边和四角不能调整窗口大小
- [ ] 没有系统标题栏及其顶部/左侧黑线
- [ ] 保留 DWM 原生窗口阴影
- [ ] 不存在自绘外层窗口阴影
- [ ] 标题、说明、状态和按钮文字全部可见
- [ ] 100%、125%、150% DPI 下布局不溢出
- [ ] 禁用和卸载插件时窗口安全关闭
- [ ] 最终 DLL 为 PE32/i386
- [ ] 无 Qt 或项目私有 DLL 依赖

> UI 验收必须结合 Bee 真实宿主中的截图或实际操作；交叉编译成功只能证明语法和链接通过，不能证明视觉与交互正确。
